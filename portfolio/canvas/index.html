Explanation

A simple library for higher order canvas functionality.

The first difference is that the origin is positioned in the center of the canvas. The positive y-direction is upwards.

Initialization

var canvas = new MyCanvas( context )
The context argument is stored inside the object. Typically you'll use 
context = document.getElementById("canvasID").getContext("2d")

Settings

Some settings like the positive y-direction can be changed. The default direction is upwards, it can be flipped to the standard canvas convention using.

MyCanvas.flipY( flip )
The argument must be of type boolean, if it is not boolean the function returns a falsy value. If it returns undefined you're playing a dangerous game since in that case you created a boolean with a value other than true/false. A succesful execution returns true. If the argument is true, the positive y-direction will be upwards. The default canvas direction requires a false argument.

Current functionality

MyCanvas.drawLine( fromx, fromy, tox, toy, [style )
Draw a line starting at (fromx, fromy) to (tox, toy)
MyCanvas.drawArrow( fromx, fromy, tox, toy, [style )
Draw an arrow starting at (fromx, fromy) to (tox, toy).
MyCanvas.drawAxes( [style )
Draw the axes in such a way that they identify the coordinate system, uses drawAxes internally.
MyCanvas.drawCircle( cx, cy, r, [style )
Draw a circle of radius r centered at (cx, cy)
MyCanvas.drawPolygon( points, [style )
Draw a general polygon starting and ending at the point defined by points[ 0 ]. Each point should be an object of the form, 
              {
                x: Number,
                y: Number
              }

Styling objects

The style argument expects a javascript object of the form 

            {
              fill: "fill color",
              stroke: "stroke color",
              width: "width of lines"
            }
At this point in time these are the only supported styling options, some options might be added later.
