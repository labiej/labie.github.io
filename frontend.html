Philosophy

Deciding whether someone is a match for your company is hard. For this reason I include this section detailing my philosophy regarding development in general. It can be summarised using the KISS principle. Why bother creating everything from scratch which inevitably leads to issues with backwards compatibility for older browsers and creates overhead when patching these issues. I also feel that the overusage of frameworks like Bootstrap and custom fonts can have some adverse effects. For simple websites it's more useful to either extract the basic grid functionality from a library or create one from scratch. The reason is quite simple, extra functionality leads to slower pages which have adverse effects in this age of instant gratification. An article that really brought this to my attention can be found on Monica Dinosaurescu's blog.

Seemingly opposite to the KISS principle is my DIY attitude. I have some ideas in the pipeline like a simple template generator where you can specify the number of columns of a straightforward website. Surely this exists in one form of another but I feel it can be made simpler and more robust if I spend some time on it. Another project is to build a standalone bookmark manager, for this I'm learning python mainly because it is so easy to build a GUI with tkinter. There exist solutions that do exactly this on windows but in general I use linux in my free time which doesn't have a native program that does this. Furthermore I want to be able to create a seamless integration with whatever browser I use at any given moment in time. This attitude shows itself in other aspects of my life as well, I enjoy gardening and carpentry. But I also do small repairs to whatever is broken myself recently I added car repairs to this list.

Finally I have to mention my drive to learn new things. This means keeping up with literature in physics but also read about websecurity specifically regarding the IoT which got some attention recently. This drive pushes me to keep up with new frameworks as well. But also to consider the impact on usability, which is something I might look into later by only using what's necessary from a framework. This will be a difficult exercise in interpreting code when and if I attempt to tackle this problem.

Languages

Nowadays web developers can use a wide variety of languages (dialects might be a better word). The basic languages are HTML5 for structuring the content, CSS for styling said content and JavaScript to introduce dynamic behaviour. Below I give an overview of the languages I'm familiar with accompanied by my level of proficiency in said language.

HTML5

CSS, dialects and frameworks

CSS can be used to achieve a lot of visual eye candy. Its latest iteration introduces a lot of interesting functionality related to animation. One aspect that's not fully supported accross different browsers are easy to use responsive design solutions. For this reasons we can use a lot of frameworks notably Twitter Bootstrap, Zurb Foundation and my favourite, Google's Materialize. This last library is based on Google's material design guidelines aimed at creating user interface for smartphone applications. As a consequence it contains a very useful color palette which are soothing on the eyes. An obvious danger is a highly uniform experience when visiting different websites. Depending on the application I use built in colors or a customized palette.

JavaScript + frameworks

Vanilla JavaScript is very powerful but becomes obtuse rather quick. For this reason I often use frameworks that simplify leveraging JavaScripts power. The most widely known example of such a framework is jQuery. jQuery by itself is big but there's an even bigger amount of plugins achieving everything from autocomplete suggestions to lightboxes. Interactivity is further enhanced using AJAX which is tedious to implement in vanilla JavaScript but incredibly easy in jQuery.

Tools

Web development evolves at an astonishing rate. From preprocessors simplifying the creation of a CSS-stylesheet to build and deployment tools. An important part of development is a version control system the most popular system is git and a prerequisite for all but the smallest projects. Other tools are needed to convert what I call dialects to languages recognized by browsers. A prime example is Sass which needs to be converted to valid CSS. Another example is JSX as used by frameworks like Facebook's React. For more complex projects one uses dependency management like gulp and grunt.
